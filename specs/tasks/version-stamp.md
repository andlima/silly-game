# Version Stamp

## Goal

Make the game's current version easily identifiable from within the game itself.
Display a version string (semver from `package.json` + build datetime) in the web
help overlay and CLI help screen.

Example output: `v0.1.0 · 2026-04-12 11:27`

## Design

### Single source of truth: `src/version.js`

A generated module that exports the version string. It is produced by a stamp
script and committed alongside code changes.

```js
// Auto-generated by `npm run stamp` — do not edit manually
export const VERSION = "v0.1.0 · 2026-04-12 11:27";
```

### Stamp script: `scripts/stamp.js`

A small Node script (no dependencies) that:

1. Reads `version` from `package.json`.
2. Formats the current local datetime as `YYYY-MM-DD HH:MM`.
3. Writes `src/version.js` with the combined string.

Register it in `package.json`:

```json
"scripts": {
  "stamp": "node scripts/stamp.js"
}
```

### Web — help overlay (`index.html`)

Add a small muted line at the bottom of `#help-content`, after the dismiss hint:

```html
<div class="help-version" id="help-version"></div>
```

CSS (alongside existing `#help-content` styles):

```css
.help-version {
  margin-top: 12px;
  font-size: 11px;
  color: #555;
  text-align: center;
}
```

In the `<script>` section, import `VERSION` from `src/version.js` and set the
element's text:

```js
import { VERSION } from './src/version.js';
document.getElementById('help-version').textContent = VERSION;
```

### CLI — help screen (`cli.js`)

Import `VERSION` and display it when the help screen is shown (alongside the
existing key-binding help text). Show it as a dim footer line, e.g.:

```js
import { VERSION } from './src/version.js';
// … in the help render section:
process.stdout.write(`\x1b[2m${VERSION}\x1b[0m`);  // dim text
```

## Changes

| File | What |
|------|------|
| `scripts/stamp.js` | **New** — stamp script |
| `src/version.js` | **New** (generated) — exports `VERSION` |
| `package.json` | Add `"stamp"` script |
| `index.html` | Import VERSION, add `.help-version` element + CSS |
| `cli.js` | Import VERSION, show in help screen |

## Verify

1. `npm run stamp` succeeds and `src/version.js` contains the expected format.
2. Opening the web game and pressing `?` shows the version string at the bottom
   of the help overlay in muted text.
3. Running `node cli.js` and opening help shows the version string.

## Notes for spec-driven changes

Any spec that bumps functionality or fixes a bug should run `npm run stamp`
**after** updating `package.json` version (if applicable) to keep the displayed
version current. Include this as a final step in the spec's verify section:

```
- Run `npm run stamp` and commit the updated `src/version.js`.
```
